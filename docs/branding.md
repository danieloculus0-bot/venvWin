# Branding Direction

## Public product naming

Do not ship the OS wrapper publicly as WinUx.

Use this split:

```text
venvWin = compatibility capsule engine
venvWin Portable = bootable portable OS shell / live USB product
WinUx = internal codename only
```

## Why

WinUx/Winux is already too close to existing public Linux distro branding. The project can still use WinUx internally as a codename, but public UI, docs, manifests, first-boot screens, and release artifacts should prefer venvWin Portable.

## Product position

Use:

```text
Portable compatibility capsule OS
```

Avoid:

```text
Windows clone
Windows skin
Windows replacement distro
```

## GUI direction

The interface may feel familiar to Windows users, but it should not impersonate Windows.

Design target:

```text
familiar desktop flow
Start-panel-like launcher
taskbar-like status strip
File Explorer-ish capsule browsing
Control Panel-ish doctor/settings sections
dark premium Linux shell
```

## Rule

Familiar-ish, not knockoff bullshit.
