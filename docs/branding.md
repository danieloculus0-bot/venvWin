# Branding Direction

## Public product naming

Use one public product name:

```text
venvWin Portable
```

Use one subsystem name:

```text
venvWin
```

Do not ship, document, or present the portable OS under alternate product names.

## Why

The product needs one clean name for users, artifacts, first-boot screens, manifests, help text, and release notes. Alternate names make the project look unfinished and confuse the flash-ready gate.

## Product position

Use:

```text
Portable compatibility capsule OS
```

Avoid:

```text
Windows clone
Windows skin
Windows replacement distro
```

## GUI direction

The interface may feel familiar to Windows users, but it should not impersonate Windows.

Design target:

```text
familiar desktop flow
Start-panel-like launcher
taskbar-like status strip
File Explorer-ish capsule browsing
Control Panel-ish doctor/settings sections
dark premium Linux shell
```

## Rule

Familiar-ish, not knockoff bullshit.
